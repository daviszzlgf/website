模块概述

学习目标

  • 知识目标：
    1. 掌握浮动（float）和定位（position）属性的工作原理与应用场景；
    2. 理解清除浮动的方法及z-index属性的作用。
  • 能力目标：
    1. 能够使用浮动实现多栏布局；
    2. 能够使用定位实现元素的精确定位与叠加效果；
    3. 能够分析并解决常见的布局兼容性问题。
  • 素养目标：
    1. 培养严谨、细致的代码调试习惯，树立规范编码与团队协作意识。

建议学时

18学时

浮动+定位+综合布局

在前面的项目中，我们已经成功地为“精品课程网站”搭建了清晰的结构骨架，并为其披上了得体的“外衣”。然而，一个精美的网页并非这些“盒子”的简单堆砌，如何将它们有机地、精准地排列在浏览器这个“舞台”上，构建出既符合设计美学又便于用户浏览的版面，是网页从“零件”走向“成品”的关键一步。

本模块中，我们将深入CSS布局的核心战场，掌握指挥这些“盒子”排兵布阵的两大经典“兵法”：浮动（Float）与定位（Position）。我们将学习如何让多个内容区块并肩而立，形成灵活的多栏布局；如何将重要的导航元素“钉”在屏幕的特定位置；以及如何处理布局中常见的“高度塌陷”等意外状况。最终综合运用这些布局技术，为我们的网站搭建起坚实、灵活且专业的页面骨架。

任务列表

  • 任务5.1：初步了解浮动特性

    基础

    在网页设计中，单纯的文字段落可能显得单调乏味。为了使内容更生动、布局更紧凑，常常需要实现图片与文字的结合，例如让一段文字自然地环绕在图片周围。本任务通过对图片添加一行简单的CSS样式，观察页面布局发生的奇妙变化——文字会自动环绕图片排列，直观地认识CSS中“浮动”属性的基本作用和初步特性。

    任务分析：

    默认布局中块级元素独占一行。浮动的作用：对图片应用float属性后，它会向左或向右移动，后方未设置浮动的文本内容会自动向上移动填补空缺，形成“环绕”。

    知识链接：浮动（float）属性

    float属性最初用于实现文本环绕图片的效果，后被广泛应用于页面布局。常用值：left（向左浮动）、right（向右浮动）、none（默认不浮动）。浮动元素会脱离正常文档流，沿指定方向浮动，直至碰到父级边框或另一个浮动框的边缘。

    正常文档流：块级元素独占一行从上到下排列，行内元素从左到右排列。脱离文档流后，元素不再遵循默认排列规则，其原位置不再保留。

    案例5.1：图文混排（图片向左浮动）
    <!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>图文混排初体验</title>
    <style>
        .container {
            width: 400px;
            border: 1px solid #ddd;
            padding: 15px;
        }
        img {
            border: 3px solid blue;
            margin-right: 20px;
            float: left;   /* 关键：向左浮动 */
        }
    </style>
</head>
<body>
    <div class="container">
        <img src="./image/img5-1.jpg" alt="示例图片" width="150">
        <p>通过本课程的学习，学生将能够独立设计并开发具有专业水准的静态网站，理解企业级开发流程和规范，为后续学习高级前端框架和全栈开发打下坚实基础。</p>
    </div>
</body>
</html>

    试一试

    修改img的float值为right，观察文字环绕方向变化。再尝试将多个块级元素同时设置浮动（左浮动或右浮动），观察它们是如何并排显示的。

  • 任务5.2：浮动的负面影响与清除浮动

    核心

    当父容器内的所有子元素都浮动脱离文档流后，未被撑开的父容器会高度塌陷至零，导致背景、边框等失效。此外，浮动元素也可能与其后的非浮动元素发生覆盖重叠。本任务通过实践观察这些“副作用”，并掌握三种清除浮动的核心方法。

    任务分析：

    关键点是通过实践观察浮动带来的负面作用，然后学习并应用三种清除浮动的方法：额外标签法（clear:both）、父元素overflow法（overflow:hidden）、伪元素清除法（推荐）。

    知识链接：

    父元素高度塌陷：父元素未设置高度时，内部如果全是浮动元素，父元素高度会变为0。
    布局错乱：非浮动元素会上移与浮动元素重叠。
    清除浮动的方法：

    • 额外标签法：在浮动元素末尾加空div并设置clear:both。
    • 父元素overflow:hidden：触发BFC，自动计算高度。
    • 伪元素法：.clearfix::after { content: ""; display: block; clear: both; }（全局复用，最常用）
    案例5.7：伪元素清除浮动（推荐）
    <!DOCTYPE html>
<html lang="zh-CN">
<head>
    <style>
        .clearfix::after {
            content: "";
            display: block;
            clear: both;
        }
        .card {
            float: left;
            width: 30%;
            margin-right: 5%;
            background: #3498db;
            color: white;
            padding: 20px;
        }
        .container {
            background: #ecf0f1;
            padding: 10px;
        }
    </style>
</head>
<body>
    <div class="container clearfix">
        <div class="card">卡片1</div>
        <div class="card">卡片2</div>
        <div class="card" style="margin-right:0;">卡片3</div>
    </div>
</body>
</html>

    试一试

    分别用额外标签法和overflow:hidden解决父元素高度塌陷问题，观察三种方法的异同。

  • 任务5.3：使用浮动实现左右两栏布局

    核心

    将浮动布局技术应用到实际的页面结构中：构建一个占据页面25%宽度的左侧边栏和一个占据75%宽度的右侧主体内容区。要求使用float属性实现两个区块的水平并排，并运用清除浮动方法确保父容器不发生“高度塌陷”。

    任务分析：

    规划两个区块：左侧边栏(.sidebar)左浮动，宽度25%；右侧主体(.main-content)左浮动，宽度75%。为父容器添加clearfix类清除浮动。

    案例5.7变体：两栏浮动布局
    <!DOCTYPE html>
<html>
<head>
    <style>
        .clearfix::after { content: ""; display: block; clear: both; }
        .sidebar {
            float: left;
            width: 25%;
            background: #e67e22;
            height: 200px;
            color: white;
        }
        .main-content {
            float: left;
            width: 75%;
            background: #2ecc71;
            height: 200px;
            color: white;
        }
    </style>
</head>
<body>
    <div class="container clearfix">
        <div class="sidebar">侧边栏</div>
        <div class="main-content">主体内容</div>
    </div>
</body>
</html>

    试一试

    尝试修改宽度比例为30%和70%，并利用浮动实现三栏布局（左、中、右）。可参考素材“案例5.8：实现三栏布局.html”。

  • 任务5.4：了解定位的基本知识

    基础

    学习CSS的position定位属性，通过实践掌握relative（相对定位）、absolute（绝对定位）和fixed（固定定位）的核心原理与应用场景。

    任务分析：

    理解三种定位模式的差异：相对定位相对于自身原始位置偏移，不脱离文档流；绝对定位相对于最近已定位祖先元素，脱离文档流；固定定位相对于浏览器视口，不随滚动移动。熟练运用top/left/right/bottom及z-index管理层叠。

    知识链接：

    position属性值： static(默认)、relative、absolute、fixed、sticky。
    z-index：控制定位元素沿Z轴的层叠顺序，数值越大越靠上。仅对定位元素有效。

    案例5.9：定位基础属性演示
    <!DOCTYPE html>
<html>
<head>
    <style>
        .box { width: 200px; height: 100px; color: white; padding: 10px; }
        .box1 { background: #3498db; position: relative; top: 20px; left: 20px; }
        .box2 { background: #e74c3c; position: absolute; top: 50px; left: 50px; }
        .box3 { background: #2ecc71; position: fixed; top: 100px; right: 100px; }
        .container { position: relative; height: 200px; background: #ecf0f1; margin-top: 30px; }
    </style>
</head>
<body>
    <div class="container">
        <div class="box box1">相对定位</div>
        <div class="box box2">绝对定位(相对container)</div>
    </div>
    <div class="box box3">固定定位</div>
</body>
</html>

    试一试

    修改定位类型为sticky（粘性定位），设置top:0，观察滚动时元素的变化。调整z-index值，观察层叠顺序。

  • 任务5.5：通过定位实现固定导航栏

    应用

    将一个普通的导航栏改造成始终可见的固定导航栏（不随页面滚动而移动），并确保不同浏览器下的视觉效果和兼容性。同时对比固定定位和粘性定位的差异。

    任务分析：

    使用position:fixed或position:sticky实现固定效果。固定定位需要为下方内容预留空间（padding-top），否则会被遮挡；粘性定位在触发前占据文档流，无需预留。

    案例5.10：固定导航栏 (fixed)
    <!DOCTYPE html>
<html>
<head>
    <style>
        body { margin: 0; }
        header {
            position: fixed;
            top: 0;
            width: 100%;
            height: 60px;
            background: #2c3e50;
            color: white;
            text-align: center;
            line-height: 60px;
            z-index: 1000;
        }
        main {
            margin-top: 80px; /* 避免被导航栏遮挡 */
            padding: 20px;
            height: 1500px;
            background: #ecf0f1;
        }
    </style>
</head>
<body>
    <header>固定导航栏</header>
    <main>页面主体内容，向下滚动查看导航是否固定</main>
</body>
</html>

    试一试

    将position:fixed改为position:sticky，观察是否需要margin-top。并使用固定定位在页面右下角实现一个“返回顶部”按钮。

  • 任务5.6：常见网页布局方式

    综合

    理解并掌握常见的国字型、T字型、三字型布局的理论结构、视觉特点与适用场景。分析每种布局在“精品课程站点”不同页面的适用可能性，并实现简易布局。

    知识链接：

    国字型：顶部通栏页眉，底部通栏页脚，中间左侧/右侧边栏+主内容，形似“国”。
    T字型：顶部通栏，下方左侧边栏+右侧主内容，无底部通栏，形似“T”。
    三字型：从上到下多个通栏堆叠，无侧边栏，形似“三”。

    案例5.11：国字型布局（Flex实现）
    <!DOCTYPE html>
<html>
<head>
    <style>
        * { margin:0; padding:0; box-sizing:border-box; }
        body { min-height:100vh; display:flex; flex-direction:column; }
        .header { height:80px; background:#ff6b6b; color:white; text-align:center; line-height:80px; }
        .main-container { display:flex; flex:1; }
        .sidebar { width:200px; background:#4ecdc4; color:white; text-align:center; padding-top:20px; }
        .content { flex:1; background:#45b7d1; color:white; text-align:center; padding-top:20px; }
        .right-sidebar { width:200px; background:#96ceb4; color:white; text-align:center; padding-top:20px; }
        .footer { height:80px; background:#feca57; text-align:center; line-height:80px; }
    </style>
</head>
<body>
    <div class="header">页眉</div>
    <div class="main-container">
        <div class="sidebar">左侧边栏</div>
        <div class="content">主体内容</div>
        <div class="right-sidebar">右侧边栏</div>
    </div>
    <div class="footer">页脚</div>
</body>
</html>

    试一试

    修改代码，实现T字型布局（去掉右侧边栏和底部通栏，页脚信息放在主体内），并使用Grid布局重新实现国字型。

归纳与总结

本项目深入探讨了实现网页布局的两大核心技术：浮动布局与定位布局。浮动布局通过 float 属性实现元素排列，常用于多栏结构，但需注意清除浮动以避免布局错乱。定位布局则通过 position 属性进行精确控制，分为相对定位、绝对定位和固定定位，分别适用于微调、脱离文档流的精准定位以及固定于视口的元素。

掌握这两种技术的核心在于明确其应用场景与特性：浮动适用于流式内容环绕，而定位适用于叠加与精准控制。务必理解绝对定位的基准点选择以及 z-index 的生效条件，这是构建稳定、符合预期页面结构的关键，也为学习更现代的布局方式奠定了坚实基础。

习题

一、单项选择题（10题）

  1. 在CSS中，设置float: left; 属性后，该元素会（ ）
    A. 向左移动 B. 脱离标准文档流，并向左浮动 C. 无变化 D. 脱离标准文档流，并向右浮动
  2. 当一个元素设置了position: absolute; 后，其定位的参考基准是（ ）
    A. 浏览器窗口的左上角 B. 其自身在文档流中的原始位置 C. 离它最近的、设置了定位（position值非static）的祖先元素 D. 页面的<body>元素的左上角
  3. 希望某个元素始终固定在浏览器窗口的顶部，不随页面滚动而移动，应使用（ ）
    A. position: static; B. position: relative; C. position: absolute; D. position: fixed;
  4. 使用 clear: both; 属性的主要作用是（ ）
    A. 清除元素的内外边距 B. 清除左右两侧浮动元素对当前元素的影响 C. 使元素变为块级元素 D. 隐藏元素
  5. 如果一个父元素的所有子元素都设置了浮动，则该父元素可能会出现（ ）问题。
    A. 外边距合并 B. 高度塌陷 C. 宽度自适应失效 D. 边框样式丢失
  6. 设置 position: relative; 的元素，其特点是（ ）
    A. 脱离文档流，不保留原始空间 B. 不脱离文档流，相对于浏览器窗口定位 C. 脱离文档流，相对于父元素定位 D. 不脱离文档流，相对于其自身原始位置进行偏移
  7. 在CSS中，用于控制定位元素层叠顺序的属性是（ ）
    A. display B. overflow C. z-index D. vertical-align
  8. 为元素设置 position: absolute; top: 50%; left: 50%; 会使该元素（ ）
    A. 水平垂直居中于其定位的参考元素 B. 左上角顶点定位在参考元素的中心点 C. 中心点定位在参考元素的左上角顶点 D. 右下角顶点定位在参考元素的中心点
  9. 下列清除浮动的方法中，推荐使用的是（ ）
    A. 为父元素设置一个固定的高度 B. 在浮动元素后添加一个空<div>并设置clear:both; C. 为父元素设置overflow:hidden; D. 为父元素添加::after伪元素并设置clear:both;
  10. 关于 z-index 属性，下列说法正确的是（ ）
    A. 任何元素都可以通过设置 z-index 来调整层叠顺序 B. z-index 的值可以是负数 C. z-index 值越大，元素在页面的位置越靠前 D. z-index 属性对 position: static; 的元素同样有效

  3. 如果希望一个绝对定位的子元素相对于其父元素进行定位，应该为父元素设置 position: __________。

三、判断题

  1. （ ）浮动元素会完全脱离文档流，不再占据任何空间。
  2. （ ）position: relative; 的元素可以通过 top, left 等属性移动，但其原始位置会被保留。
  3. （ ）z-index 的属性值越大，元素在层叠顺序中就越靠下。
  4. （ ）使用 float 可以实现多栏布局，例如一个两栏布局。
  5. （ ）绝对定位（absolute）和固定定位（fixed）都会使元素脱离标准文档流。
