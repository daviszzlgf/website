模块7：多媒体嵌入

  • 首页

  • 模块7：多媒体嵌入

模块概述

学习目标

  • 知识目标：了解网页中常用的音视频格式及其兼容性；掌握HTML5 <audio>、<video> 标签及常用属性的用法；掌握CSS美化音频、视频控件的方法。
  • 能力目标：能够使用audio标签和video标签在网页中插入音频、视频文件，并根据需求配置基础属性；能使用CSS样式简单美化音视频播放器，实现与页面设计风格的统一；能独立制作包含音视频的简单网页。
  • 素养目标：树立用技术传播正向价值的意识，培养对网络空间责任的初步认知；树立多媒体资源的版权保护意识，规范使用正版或开源授权的音视频素材。

建议学时

4学时

理论+实践

在互联网从“文字时代”迈向“视听时代”的进程中，多媒体已成为现代网页的核心组成部分。它打破了纯文本的单调局限，通过图像、音频、视频、动画等元素，让网页从信息载体升级为沉浸式交互空间。本章将深入解析网页多媒体的应用逻辑，助你掌握用多媒体赋能网页、打造高吸引力交互场景的实用技能。

任务列表

  • 任务7.1：在网页中嵌入视频与音频

    核心

    本任务将通过实战操作，掌握 HTML5 嵌入多媒体的核心方法。首先需准备合法的音频文件（如MP3格式）和视频文件（如MP4格式），然后分别使用<audio>标签和 <video> 标签完成音视频的嵌入，并配置基础播放功能（如显示控件、设置视频封面等），最终实现音视频在浏览器中的正常加载与播放。

    任务分析：

    不同浏览器对音视频格式的支持存在差异（如Firefox对WebM支持更好，IE对MP4支持有限），需理解格式选择的优先级。同时需注意音频自动播放策略、视频封面设置等细节。

    常用音视频格式及兼容性

    MP3
    有损压缩 · 兼容100% · 网页首选
    WAV
    无损格式 · 体积大 · 适合短音效
    MP4
    通用视频 · 兼容100% · H.264编码
    WebM
    开源 · 体积小 · IE/Safari不支持

    <audio> 核心属性

    controls

    显示浏览器默认的音频控件（播放/暂停、音量调节、进度条）。

    loop

    设置循环播放。

    preload

    预加载方式：auto(全部)、metadata(元数据)、none(不预加载)。

    autoplay

    自动播放（通常需配合muted使用）。

    <video> 核心属性

    poster

    指定视频播放前显示的封面图路径。

    muted

    默认静音播放，常与autoplay搭配。

    width/height

    设置播放器尺寸，建议只设一个避免拉伸。

    <source>

    提供多种格式文件，提升兼容性。

    示例代码：嵌入音频与视频
    <!-- 音频：单一格式 -->
<audio src="audio/music.mp3" controls loop>
    您的浏览器不支持音频播放，请升级浏览器！
</audio>

<!-- 音频：多格式兼容 -->
<audio controls>
    <source src="audio/music.mp3" type="audio/mpeg">
    <source src="audio/music.ogg" type="audio/ogg">
    您的浏览器不支持音频播放，请升级浏览器！
</audio>

<!-- 视频：带封面、自动静音播放 -->
<video controls width="600" poster="images/cover.jpg" autoplay muted>
    <source src="video/movie.mp4" type="video/mp4">
    <source src="video/movie.webm" type="video/webm">
    您的浏览器不支持视频播放，请升级浏览器！
</video>

    试一试：尝试制作自己的博客主页，并为其添加背景音乐。

  • 任务7.2：使用CSS定制多媒体播放器样式

    美化

    浏览器默认的音视频控件样式往往无法匹配页面设计风格。本任务需通过CSS修改播放器的核心视觉元素，包括音频进度条颜色、视频播放按钮样式、控制栏背景色等，实现“自定义风格与页面统一”的多媒体播放器。

    任务分析：

    不同的网页有不同的设计风格，播放器的样式也应该与之匹配。我们可以修改播放器的颜色、大小等外观。音频主要调整播放按钮；视频可修改控制栏背景色、播放/暂停按钮、封面图显示效果。注意不同浏览器对伪元素的支持差异，必要时隐藏原生控件自定义。

    常用CSS伪元素（WebKit内核）

    ::-webkit-media-controls

    整个控件容器，可修改背景色、内边距、圆角。

    ::-webkit-media-controls-play-button

    控制播放/暂停按钮样式（背景、圆角、大小）。

    ::-webkit-media-controls-volume-slider

    音量调节滑块样式。

    ::-webkit-media-controls-fullscreen-button

    全屏按钮样式（仅视频）。

    浏览器兼容性提示

    Chrome/Edge
    良好支持

    -webkit- 伪元素可用，支持广泛。

    Firefox
    部分支持

    部分伪元素可能失效，建议使用自定义控件方案。

    Safari
    支持-webkit-

    与Chrome类似，但部分细节有差异。

    示例代码：CSS美化音视频控件
    /* 音频播放器宽度 */
.my-audio {
    width: 500px;
    height: 40px;
}

/* 播放按钮改为蓝色圆形 */
.my-audio::-webkit-media-controls-play-button {
    background-color: #2196F3;
    border-radius: 50%;
    padding: 10px;
}

/* 视频控制栏半透明背景 */
video::-webkit-media-controls {
    background-color: rgba(0, 0, 0, 0.5);
    border-radius: 5px;
}

/* 播放按钮橙色，悬浮加深 */
video::-webkit-media-controls-play-button {
    background-color: #ff6600;
    border-radius: 50%;
    padding: 10px;
}
video::-webkit-media-controls-play-button:hover {
    background-color: #cc5200;
}

/* 进度条已播放部分 */
video::-webkit-media-controls-played-progress {
    background-color: #ff6600;
}

    试一试：拍摄一段视频，写一篇心情日记，添加到自己的博客，并用自定义样式美化。

实训任务

实训7.1：太行山四日游多媒体页面制作

综合

综合运用模块7所学，制作一个介绍太行山四日游的多媒体页面，包含音频背景、视频日志、图文展示等。

任务要求：

  • 页面包含精彩瞬间、每日行程、旅行小贴士、推荐景点四个版块。
  • 在页面开篇嵌入背景音乐（MP3），第2日行程嵌入太行山视频（MP4/WebM兼容）。
  • 精彩瞬间使用flex布局展示至少3张图片。
  • 每日行程使用flex实现图片与文字左右排列（图片45%，文字50%）。
  • 旅行小贴士用列表展示5条实用信息（温差、装备、讲解、餐饮、季节）。
  • 推荐景点以卡片形式展示3个景点（图片+标题+推荐语）。
  • 使用CSS对音频/视频播放器进行简单美化（如按钮颜色、控制栏背景）。

详细实现可参考教材示例代码，完成后在Chrome和Firefox中测试兼容性。

归纳与总结

本章聚焦网页多媒体应用，系统学习了音频、视频在网页中的嵌入与美化。我们先了解 MP3、MP4 等常用音视频格式及浏览器兼容性差异，明确格式选择优先级；随后掌握<audio>和<video>标签用法，能灵活配置src、controls、loop、poster等核心属性，实现音频、视频播放控制、视频封面显示等基础功能。通过“多媒体网页”项目实战，完成了素材准备、基础HTML结构创建、音视频嵌入与多浏览器兼容性测试，还学会用CSS伪元素定制播放器样式，修改进度条、控制栏、按钮等视觉元素以匹配页面风格，同时树立了技术正向传播意识与多媒体版权保护意识，为后续开发更丰富的多媒体网页筑牢基础。

拓展练习

修复“产品介绍页”多媒体故障：

某开发者制作的“智能手环产品介绍页”存在3处问题：1. 视频在Firefox中无法播放；2. 音频设置了autoplay但未自动播放；3. 视频封面图未显示且控制栏样式与科技蓝主题不匹配。请根据需求修复，并提供兼容多种格式、自动播放策略及CSS美化方案。

原代码片段：

待修复代码
<!-- 视频区域 -->
<div class="product-video">
    <video src="video/bracelet-intro.mp4" controls width="800">
        您的浏览器不支持视频播放！
    </video>
</div>
<!-- 音频区域 -->
<div class="product-audio">
    <audio src="audio/product-intro.mp3" autoplay>
        您的浏览器不支持音频播放！
    </audio>
</div>

修复要求：解决Firefox视频播放（添加WebM格式）；音频自动播放（考虑 muted 或用户交互）；添加视频封面图，控制栏背景改为 rgba(22, 93, 255, 0.8)，进度条已播放部分改为 #165DFF。

本章习题（选择题/填空题）

  • 以下哪种视频格式是网页中的“通用格式”，支持所有现代浏览器？ A. WebM B. MP4 C. OGV D. FLV （答案：B）
  • 在HTML5的<audio>标签中，用于显示浏览器默认播放控件的属性是？ A. show-controls B. controls C. display-controls D. play-controls （答案：B）
  • 若需让视频在播放前显示一张封面图，应使用<video>标签的哪个属性？ A. cover B. img C. poster D. thumbnail （答案：C）
  • 关于OGG音频格式，正确的是？ A. 兼容100% B. 开源有损，IE/Safari不支持 C. 无损格式 D. 网页首选 （答案：B）
  • 设置了<video autoplay>但未自动播放，最可能的原因是？ A. 未添加controls B. 未配合muted C. 路径错误 D. 浏览器不支持 （答案：B）

填空题：1. <audio>的preload属性可选值有auto、metadata和none。2. 网页视频格式中兼容性差、逐渐被WebM替代的是OGV；音频格式兼容100%的是MP3。3. 避免视频拉伸，建议只设置width或height中的一个。4. 浏览器不支持音视频时，可在标签内添加备用文本/降级提示。
