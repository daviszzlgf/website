模块概述

学习目标

  • 知识目标：理解网页、网站、浏览器及HTTP/HTTPS、URL等基本概念及关系。掌握HTML与CSS在网页构建中的作用及Web标准的基本要求。熟悉VS Code开发环境的基本操作与配置方法。了解网页设计与制作的基本流程。
  • 能力目标：能独立完成编码环境的搭建与基础配置。能创建符合规范的初始HTML文件并预览。
  • 素养目标：树立用技术传播正向价值的意识，培养对网络信息责任的初步认知。编写代码时，遵循标准性，注重可读性与可维护性。

模块时长

4学时

基础模块

在当今社会，互联网已成为不可或缺的基础设施，全球数十亿用户通过网页获取信息、交流互动、办理事务。网页不仅是信息载体，更是进行购物消费、参与社交互动的重要入口。优秀的网页设计融合了技术实现与视觉艺术，是连接用户与数字世界的桥梁。

在我国建设网络强国的进程中，网页设计与制作相关人才的需求日益增长。本课程以“精品课程站点”项目为载体，系统讲解网页设计与制作的基础知识，为从事相关岗位和继续学习打下基础。

    知识链接

    网页(Web Page)

    网页是构成网站的基本元素，是一个包含HTML标签的纯文本文件，可以包含文字、图片、声音、视频等内容元素，是承载各种网站应用的平台。

    静态网页：静态网页是标准的HTML文件，它的文件扩展名是.htm、.html等，可以包含文本、图像、声音、动画、客户端脚本等，是网站建设的基础。但是静态网页是没有后台数据库和服务器端程序的网页。

    动态网页：动态网页是结合了HTML以外的高级程序设计语言和数据库技术进行的网页编程技术生成的网页。动态网页一般是以数据库技术为基础，可以与后台数据库进行数据交互与数据传递，能够大大减少维护网站的工作量。常见动态网页文件以.aspx、.asp、.jsp、.php、.perl、.cgi等为后缀。

    网站(Website)

    网站是指在互联网上，根据一定的规则，使用HTML等工具制作的用于展示特定内容相关网页的集合。网站拥有域名或地址并提供一定网络服务的主机，是存储文件的空间，以服务器为载体。人们可通过网页浏览器（Web Browser）等进行访问、查找文件，也可通过远程文件传输方式上传、下载网站文件。

    网站采用动态网页还是静态网页主要取决于网站的功能需求和网站内容的多少，如果网站功能比较简单，内容更新量不是很大，采用纯静态网页的方式会更简单，反之一般要采用动态网页技术来实现。动态网站也可以采用静动结合的原则。

    其他相关概念

    浏览器(Browser)：浏览器是用来检索、显示以及传递Web信息资源的应用程序。常见的浏览器有Microsoft Edge、Chrome、Firefox、Safari等。

    URL (Uniform Resource Locator)：URL是统一资源标识符，描述了一个网页在网络服务器上的特定位置，即网址。URL通常由四个部分组成：协议、服务器名称（或IP地址）、路径和文件名。

    HTTP（Hyper Text Transfer Protocol）：HTTP是超文本传输协议, 是一个简单的请求-响应协议，是通过互联网传输文档的数据传送协议，规定了浏览器和互联网服务器之间的通信规则。

    HTTPS（Hyper Text Transfer Protocol Secure）：HTTPS是以安全为目标的 HTTP 通道，在HTTP的基础上通过传输加密和身份认证保证了传输过程的安全性。

    IP地址：IP地址是互联网协议地址提供的一种统一的地址格式，它为互联网上的每一个网络和每一台主机分配一个逻辑地址，以此来屏蔽物理地址的差异。

    域名：域名是由一串用点分隔的名字组成的互联网上某一台计算机或计算机组的名称，用于在数据传输时对计算机的定位标识。通过域名系统（DNS）将域名和IP地址相互映射。

    超链接：超链接全称超文本链接，是通过非线性方式组织信息，以锚点链接同一文档或不同服务器资源，实现目标内容的自由跳转。超链接分为文本超链接与图像超链接，按路径可分为内部链接、锚点链接和外部链接。

    步骤3：探究网页浏览器的作用

    用户在浏览器中输入网址，信息通过互联网传输后，能在浏览器中呈现指定的网页。首先浏览器将网址通过互联网发送到对应的网站服务器，Web服务器处理后返回相应数据。浏览器下载数据后开始解析网页源文件（包含HTML标签的纯文本文件），按照HTML语法规则排版，最终显示页面。同时浏览器在解析过程中会分析超链接并显示。浏览器的作用就是负责解释网页HTML源文件的语法并展示页面内容。在制作网站时，通常先在本地计算机上进行设计和测试，例如在浏览器地址栏输入“127.0.0.1/main.html”可访问本地的精品课程网站首页。

    知识链接

    HTML（Hyper Text Markup Language）

    HTML（超文本标记语言）是一种建立网页文件的语言，通过标记式的指令(Tag)，将文字、图形、动画、声音、表格、链接、影像等内容显示出来。HTML是由万维网发明者蒂姆·伯纳斯·李于1990年创立，2014年W3C发布HTML5。HTML使用标签来规定网页中各元素的属性和位置，标签通常成对出现，如<html>...</html>。一个标签可以有一个或多个属性，如<html lang="zh-CN">。

    Web标准

    Web标准是由W3C等组织制定的技术规范集合，旨在统一网页开发与呈现。网页主要由三部分组成：结构（Structure）、表现（Presentation）和行为（Behavior），分别对应结构化标准语言（HTML5等）、表现标准语言（CSS）以及行为标准（ECMAScript、DOM）。这三要素分离，确保代码可维护性。

    CSS (Cascading Style Sheets)

    CSS（层叠样式表）是用于控制网页样式和布局的语言。它不仅能静态地修饰页面，还能动态地控制元素。CSS的核心特点是“层叠性”，允许样式继承和叠加。目前主流版本CSS3将功能划分为多个模块，如选择器、动画、Flexbox和Grid等。CSS样式规则由选择器和声明块组成，例如：header { background: #f0f0f0; text-align: center; }。

    网站开发流程

    网站开发是一个系统化的过程，行业标准有以下五个核心阶段：

    规划与分析：明确网站目标、用户与功能，制定项目计划、站点地图，完成技术选型。主要参与者：产品经理、客户。

    设计与原型：将规划转化为具体视觉表现，产出视觉稿、交互原型和设计风格指南。主要参与者：UI/UX设计师。

    开发与实现：前端使用HTML/CSS/JavaScript构建用户界面；后端处理服务器与数据库逻辑；前后端联调。主要参与者：前端/后端工程师。

    测试与部署：进行全面测试（功能、兼容性、性能、安全等），部署至线上服务器。主要参与者：测试工程师、运维与开发工程师。

    维护与迭代：包括内容更新、功能优化与迭代开发。主要参与者：运营、运维与开发工程师。

    网站代码编辑器与集成开发环境

二、名词解释

1. 网页

网页是构成网站的基本元素，是一个包含HTML标签的纯文本文件，可以包含文字、图片、声音、视频等内容元素，是承载各种网站应用的平台。

2. 网站

网站是指在互联网上，根据一定的规则，使用HTML等工具制作的用于展示特定内容相关网页的集合。网站拥有域名或地址并提供一定网络服务的主机，是存储文件的空间。

3. HTTP协议

HTTP是超文本传输协议，是一个简单的请求-响应协议，是通过互联网传输文档的数据传送协议，规定了浏览器和互联网服务器之间的通信规则。

4. URL

URL是统一资源标识符，描述了一个网页在网络服务器上的特定位置，即网址。通常由协议、服务器名称（或IP地址）、路径和文件名组成。

5. HTML

HTML（Hyper Text Markup Language）是超文本标记语言，是一种建立网页文件的语言，通过标记式的指令(Tag)将文字、图形、动画等内容显示出来。

6. CSS

CSS（层叠样式表）是用于控制网页样式和布局的语言。它不仅能静态地修饰页面，还能动态地控制元素，实现精确排版和视觉效果。
